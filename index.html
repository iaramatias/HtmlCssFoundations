Introdução para Estudante de Programação

**** Porque estudar HTML ****

Para dominar o HTML e construir uma base sólida para o seu futuro no PHP e no Git, você precisa entender que ele não é apenas uma lista de comandos, mas a linguagem de marcação que define o significado e a estrutura da web.

Aqui está um guia detalhado do que vamos estudar:

  • A Mentalidade de "Caixas" (Estrutura e Hierarquia)
  • Anatomia de uma Tag e Atributos
  • Semântica: O "Cérebro" do HTML
  • O Coração da Integração: Formulários
  • Caminhos de Arquivo (Relativo vs. Absoluto)

1. A Mentalidade de "Caixas" (Estrutura e Hierarquia)

Tudo no HTML é uma "caixa". Uma tag abre, contém algo e fecha. Entender a hierarquia é fundamental para não "quebrar" o site quando você começar a inserir lógica PHP no meio.

Tags de Bloco (block): Ocupam toda a largura da linha (ex: <div>, <h1>,<p>, <ul>)

Tags de Linha (inline): Ocupam apenas o espaço do conteúdo (ex:<a>,<span>,<strong>)

Aninhamento: Você nunca deve cruzar tags. O correto é fechar a última que abriu primeiro:<div><p>Texto</p></div>

2. Anatomia de uma Tag e Atributos

As tags podem ter "instruções extras" chamadas atributos.

  • id: Um identificador único para um elemento (como um CPF).
  • class: Um identificador para vários elementos (como uma farda de escola).
  • src e href: Atributos de origem e destino (essenciais para imagens e links).
  • style: Onde você coloca o CSS diretamente na tag (estilo inline), como você fez para centralizar o texto.

3. Semântica: O "Cérebro" do HTML

Hoje em dia, não basta o site "parecer" certo; ele precisa "falar" o que é. Usar as tags corretas ajuda o Google (SEO) e tecnologias assistivas para cegos.

Em vez de usar <div> para tudo, use:

  • <header> para o topo.
  • <nav> para menus de navegação.
  • <main> para o conteúdo principal.
  • <footer> para o rodapé.
  • <section> e <article> para blocos de conteúdo.

4. O Coração da Integração: Formulários (<form>)

Como você quer aprender PHP, esta é a parte mais importante. O HTML coleta os dados, e o PHP os processa. Você deve dominar:

  • Atributo action: Para qual arquivo PHP os dados serão enviados.
  • Atributo method: GET (dados na URL) ou POST (dados escondidos/seguros).
  • Tipos de Input: text, password, email, submit, checkbox.
  • O Atributo name: Sem o name, o PHP não consegue "pescar" o valor que o usuário digitou.

5. Caminhos de Arquivo (Relativo vs. Absoluto)

Você precisa entender como o navegador localiza as coisas.

  • Caminho Absoluto: Um link completo da internet (https://google.com).
  • Caminho Relativo: Onde o arquivo está em relação ao seu index.html.
  • imagem.jpg (está na mesma pasta).
  • fotos/perfil.jpg (está dentro de uma pasta chamada fotos).

6. O que o HTML NÃO faz (Limitações)

  • HTML define O QUE é o conteúdo (Estrutura).
  • CSS define COMO ele aparece (Estética/Beleza).
  • PHP define O QUE ELE FAZ (Inteligência/Banco de Dados).
  • Git define COMO ELE É SALVO (Histórico/Versões).
